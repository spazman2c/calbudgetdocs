Accounts and categories make the calendar readable.

## Accounts

An account is a forecast container. Most users start with one checking account.

Account fields include:

- Name
- Starting balance
- Account type
- Color
- Optional bank name
- Optional last four digits
- Brand color preference

## Multiple accounts

Plus and Advisor users can create unlimited accounts. Free users are limited to one account.

Multiple accounts are useful when:

- Bills and spending live in separate checking accounts.
- A user wants to plan a savings account separately.
- A couple tracks shared and personal accounts.
- Debt payments need a dedicated forecast.

## Categories

Categories label transactions and control how the amount affects the forecast.

| Category type | Effect |
| --- | --- |
| Income | Increases the balance. |
| Expense | Decreases the balance. |

Category colors help users scan the calendar and reports.

## Category rules

Category rules can match transaction titles and automatically suggest categories during import or reconciliation workflows.

Rules include:

- Match type
- Match value
- Optional account
- Transaction type
- Priority
- Enabled status

## Free plan limits

Free users can create up to 25 categories. Plus and Advisor remove this limit.
