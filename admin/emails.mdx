The email admin page shows platform email activity.

## Email event fields

- Recipient
- Subject
- Kind
- Status
- Scheduled time
- Sent time
- Failed time
- Attempt count
- Last error
- Resend message ID
- Delivered time
- Opened time
- Open count
- Clicked time
- Click count
- Last clicked URL

## Pagination

The email page should paginate at 10 emails per page. This keeps the admin view readable and avoids a nested scroll area.

## Retry failed emails

Failed events should have a resend option when enough data exists to retry safely.

Common failure reasons:

- Resend not configured locally
- Invalid recipient address
- Rate limit or temporary provider failure
- Missing template data

## Tracking opens and clicks

Resend webhook events update open and click fields. Tracking depends on provider support and user email client behavior.

Open tracking can be blocked by privacy features. Treat it as directional, not guaranteed.

## Newsletter controls

Newsletter management should not clutter the email log if it is not actively used. Keep the email page focused on platform email events.
