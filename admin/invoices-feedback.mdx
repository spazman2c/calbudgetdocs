The invoices and feedback pages support day-to-day operations.

## Invoices

The invoices page helps admins inspect Stripe billing details. It should load reliably and show empty states when no invoices are available.

Use invoices to answer:

- Did the customer pay?
- Which invoice is next?
- Was payment successful?
- Is the customer attached to the right Stripe record?

## Feedback

Feedback helps identify product friction.

Useful feedback tags:

- Bug
- Confusing flow
- Feature request
- Billing issue
- Import issue
- Mobile issue
- Advisor issue

## Operational loop

When feedback identifies a recurring issue:

1. Fix the product or copy.
2. Update help/docs.
3. Add an admin support script if needed.
4. Consider a changelog entry.
