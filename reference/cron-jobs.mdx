Cron routes handle recurring platform work.

## Cron route map

| Route | Purpose |
| --- | --- |
| `/api/cron/reconcile-subscriptions` | Refresh Stripe, Apple, and RevenueCat subscription state. |
| `/api/cron/drip-emails` | Process lifecycle and drip email events. |
| `/api/cron/newsletter` | Process newsletter sends when enabled. |
| `/api/cron/inactivity` | Handle inactivity-related lifecycle checks. |
| `/api/cron/advisor-checkups` | Generate and email weekly Advisor checkups. |
| `/api/cron/advisor-action-reminders` | Send reminders for due Advisor actions. |

## Operational expectations

- Cron routes should be protected.
- Jobs should be idempotent.
- Email jobs should avoid duplicate sends.
- Subscription sync should handle missed webhooks.
- Failures should be logged with enough detail to debug.

## When to run manually

Run a manual sync when:

- A user paid but still sees Free.
- RevenueCat shows a trial or cancellation not visible in admin.
- Stripe customer portal differs from local state.
- Support needs an immediate refresh.

## Advisor cadence

Advisor checkups are generated roughly weekly. Fresh checkups should not be regenerated unnecessarily unless force mode is used for testing or support.
