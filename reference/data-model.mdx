This page summarizes the core database model.

## User-owned planning data

| Table | Purpose |
| --- | --- |
| `users` | Account identity, preferences, billing fields, subscription fields, access plan, and usage. |
| `accounts` | Forecast accounts with balances, types, colors, and optional bank details. |
| `categories` | User labels for income and expenses. |
| `transactions` | Dated income and expense items, recurring metadata, notes, URLs, cleared state. |
| `bookmarks` | Quick-add transaction templates. |
| `category_rules` | Rule-based categorization for imported or reviewed rows. |

## Imports and reconciliation

| Table | Purpose |
| --- | --- |
| `import_batches` | Uploaded statement batch. |
| `import_rows` | Normalized statement rows. |
| `reconciliation_sessions` | Review session for imported rows. |
| `transaction_matches` | Suggested or accepted matches between imports and planned transactions. |

## Advisor

| Table | Purpose |
| --- | --- |
| `advisor_conversations` | Advisor chat session metadata. |
| `advisor_messages` | Advisor and user chat messages. |
| `advisor_checkups` | Generated weekly checkups. |
| `advisor_actions` | Generated action plans and reminders. |

## Billing

Subscription state is stored primarily on `users` for fast entitlement checks. Fields include Stripe, Apple, and RevenueCat identifiers, product IDs, statuses, expiration dates, and sync timestamps.

## Shared access

| Table | Purpose |
| --- | --- |
| `budget_invites` | Invite token hash, role, status, expiry, and email event. |
| `shared_budget_members` | Active membership between owner and member users. |

## Support and email

| Table | Purpose |
| --- | --- |
| `support_tickets` | User tickets and admin replies. |
| `email_events` | Outbound email log and status. |
| `resend_webhook_events` | Incoming webhook event processing. |
| `newsletter_sends` | Newsletter campaign metadata. |

## Content

| Table | Purpose |
| --- | --- |
| `blog_posts` | Blog drafts, scheduled posts, and published posts. |
| `help_articles` | Help center content and metadata. |
| `changelog_entries` | Public product changelog entries. |

## Privacy and deletion

Most user-owned planning tables cascade from `users`. Deletion flows should verify all user-owned data is removed or anonymized according to policy.
