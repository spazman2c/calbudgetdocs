## Product surfaces

CalBudget currently has four major route lanes:

| Lane | Routes | Audience | Notes |
| --- | --- | --- | --- |
| Public marketing and content | `/`, `/features`, `/blog`, `/support`, `/privacy`, `/terms`, `/changelog`, `/business` | Visitors and customers | Public SEO and conversion surfaces. |
| Consumer app | `/calendar`, `/calendar/[year]/[month]`, `/debt`, `/billing`, `/subscribe`, `/invite/[token]` | Personal budgeting customers | Uses consumer database and schema. |
| CalBudget Business / BIZ | `/business`, `/biz`, `/biz/calendar` | Verified business users and internal demos | Keep separate from consumer docs. |
| Internal admin | `/quantum-nebula-9x/*` | CalBudget admins only | Hidden, admin-guarded operational surface. |

## Consumer app routes

| Route | Purpose | Access |
| --- | --- | --- |
| `/calendar` | Redirects to the current local month. | Authenticated app |
| `/calendar/[year]/[month]` | Main cash-flow calendar. | Subscription-gated |
| `/debt` | Debt payoff planner. | Subscription-gated |
| `/billing` | Billing/subscription management surface. | Authenticated |
| `/subscribe` | Subscription gate and plan checkout entry. | Public/auth-aware |
| `/invite/[token]` | Shared-access invite acceptance. | Invite recipient |

Consumer routes use:

| Boundary | Location |
| --- | --- |
| Database env var | `DATABASE_URL` |
| Schema | `src/db/schema.ts` |
| Migrations | `src/db/migrations` |
| DB client | `src/db` |

## CalBudget Business routes

| Route | Purpose | Notes |
| --- | --- | --- |
| `/business` | Public marketing page for CalBudget Business. | Public, business lane. |
| `/biz` | Redirects to the current BIZ calendar month. | BIZ app entry. |
| `/biz/calendar` | Current BIZ calendar demo/platform view. | Admin-gated in the current platform. |

BIZ routes use:

| Boundary | Location |
| --- | --- |
| Database env var | `BIZ_DATABASE_URL` |
| Schema | `src/db/biz/schema.ts` |
| Migrations | `src/db/biz/migrations` |
| DB client | `src/db/biz` |

<Warning>
  Do not import the consumer `db` from `src/db` in BIZ code. Do not import `bizDb` from `src/db/biz` in consumer code. The docs should reinforce this split whenever they describe routes, admin workflows, or data ownership.
</Warning>

## Internal admin routes

The internal admin surface is hidden under `/quantum-nebula-9x` and guarded by admin access.

| Route | Purpose |
| --- | --- |
| `/quantum-nebula-9x` | Admin dashboard with operational metrics. |
| `/quantum-nebula-9x/users` | User list, profile edits, subscription/free-access controls, reset tools. |
| `/quantum-nebula-9x/invoices` | Stripe invoice browser and MRR estimates. |
| `/quantum-nebula-9x/emails` | Email event log, newsletter tools, and templates. |
| `/quantum-nebula-9x/blog` | DB-backed blog CRUD and publishing. |
| `/quantum-nebula-9x/support` | Support ticket queue. |
| `/quantum-nebula-9x/feedback` | Feedback campaigns and rewards. |
| `/quantum-nebula-9x/changelog` | DB-backed changelog CRUD. |
| `/quantum-nebula-9x/audit-log` | Admin action audit log. |
| `/quantum-nebula-9x/documentation` | Internal documentation/admin docs surface. |

Do not link hidden admin routes from public customer docs.
