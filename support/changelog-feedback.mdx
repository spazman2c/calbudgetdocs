CalBudget includes changelog and feedback systems to keep users informed and close the product loop.

## Changelog

The public changelog shows product updates. Admins can create and manage changelog entries from the admin panel.

Changelog entries include:

- Version
- Title
- Date
- Changes
- Creation timestamp

## Release announcement

The app can show a release announcement modal when users have not seen the latest changelog version.

## Feedback

Users can submit feedback from the app. Admins review feedback in the admin panel.

Feedback is useful for:

- Confusing workflows
- Feature requests
- Bugs
- Billing problems
- Onboarding problems

## Product loop

Use feedback to update:

- Help center articles
- Docs pages
- Onboarding copy
- Empty states
- Admin support scripts
