Shared access lets an Advisor user invite another trusted person into a budget.

## Roles

| Role | Intended access |
| --- | --- |
| Viewer | Can review the forecast and context. |
| Editor | Can help edit planning data. |
| Manager | Can manage shared access and higher-trust budget settings. |

## Invite workflow

1. Owner opens shared access.
2. Owner enters the invitee email and role.
3. CalBudget sends an invite.
4. The invitee opens the invite link.
5. The invitee signs in or creates an account.
6. Access begins after acceptance.

## Invite states

- Pending
- Accepted
- Revoked
- Expired

## Security expectations

- Invites expire.
- Owners can revoke access.
- Shared users should only receive the access level the owner chose.
- Owners should use the lowest access level that fits the relationship.

## Best practices

Use **Viewer** for a partner who only needs visibility. Use **Editor** for a trusted collaborator. Use **Manager** only when the invited person should manage access.
