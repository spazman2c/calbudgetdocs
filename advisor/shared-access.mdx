Shared access lets an Advisor user invite another person into a budget.

## Roles

| Role | Intended access |
| --- | --- |
| Viewer | Can review the forecast and context. |
| Editor | Can help edit planning data. |
| Admin | Can manage shared access and higher-trust budget settings. |

## Invite workflow

1. Owner opens shared access.
2. Owner enters the invitee email and role.
3. CalBudget creates an invite token.
4. The email system sends an invite.
5. The invitee opens the invite link.
6. The invitee signs in or creates an account.
7. Membership is created after acceptance.

## Invite states

- Pending
- Accepted
- Revoked
- Expired

## Security expectations

- Invite tokens are stored as hashes.
- Invites expire.
- Owners can revoke access.
- Access checks should verify active membership and role.
- Shared users should only see data they are allowed to access.

## Best practices

Use viewer for a partner who only needs visibility. Use editor for a trusted collaborator. Use admin only when the invited person should manage access.
